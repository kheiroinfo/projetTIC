  5. Initiation aux techniques de communication :

    6. Objectifs de communication :

    Les objectifs de communication correspondent aux réactions que l‘on attend du récepteur. Un objectif de communication doit se définir selon 3 dimensions :

    • Au niveau « cognitif » la communication a pour but La prise de conscience l’attention, la notoriété et l’information au sujet d’une marque, d’un produit d’une idée, ou d’une entreprise, à ce stade, la communication est avant tout informative.
    • Au stade « affectif » c’est de Modifier ou renforcer l’image de produit en agissant sur les attitudes ou les opinions, que ce soit pour un produit ou une marque.
    • Au stade « comportemental » le but est de modifier les comportements en provoquant l’achat, l’essai.

    • Les partenaires de communication :

    • Les vecteurs :

      3. : Il s’agit de mettre en avant les caractéristiques, les performances et les avantages du produit dans une communication crédible et cohérente.

    2. La communication de groupe :

      est le schéma utilisé lorsque vous vous adressez à vos salariés, un manager à son équipe ou bien lors de réunions. Un émetteur (à un moment donné) veut faire passer un message à plusieurs destinataires. Là encore, un retour peut être attendu, voire discuté afin de tirer le maximum des avis de chacun.

      est par exemple celle de la publicité. En général, on n’attend pas de feedback, bien que l’efficacité de cette communication puisse se mesurer par exemple en termes de ventes ou autres.

Les résaux informatiques

1-Resaux Network :
un reseaux designe un ensemble d'ordinateurs et de peripheriques connecté entre eux , qui permet d'echange de données et de ressources.
mise en reseaux:
mettre a la disposition toutes les moyens permettant de relier des ordinateurs afin de connecté a un reseau.

Evolution:
un reseau depend de deux concept de base :
-la communication : qui permet d'echange des données entre processus.
-l'interconnexion : qui assure la cirulation des données entre les noeuds.

2-les caractéristiques de base d'un reseau :
la tepologie : qui definit la structure de reseau, on designe la tepologie physique et la tepologie logique.
le débit : mesure la quantité de données echangé par seconde.
la distance maximale : qui peut separer deux appareils de reseau.
le nombre de noeuds maximum

3-quelque terminologies en resau :
station de travail : toute machine qui peut être relie a un reseau.
noeud : c'est toute station de travail qui peut être adressê par un numero unique.
paquet: le plus petit unité d'information quit peut être circulé sur le réseau.
reseau homogene : toutes les ordinateurs et les materiels de marque identiques .
reseau heterogene : toutes les ordinateurs et les matriels de constructeurs divers .
le debit : quantité de données circulé par seconde .

4-Les elements d'un reseau:
-les ordinateurs.
-les logiciels.
-les supports de LAN , WAN.
-les équipements d'interconnexion de LAN.

5-Classification de réseau :
les reseaux peuvent être classifiês en fonction de diffêrentes criteres :
5-1 par portee l'envergure : LAN , MAN , WAN.
-LAN : c'est un reseau local dit "priviés", constitué de plusieurs périphériques connecté entre eux dans des petites entreprises.
-MAN : c'est un réunion de plusieurs réseaux locaux.
-WAN : Se sont des réseaux étendus assurant la circulation des données sur des grands distances .
Il existe d'autres comme : PAN , WPAN , SAN.
5-2 L'ouverture :
selon leur utilisation on destingue :
-Intranet : reseau interne d une entreprise.
-Externet : reseau externe d une structure organisation.
-Internet , reseau public , nationaux ou internationaux.
5-3-Les support de connexion :
-Resau filaires: Le réseau filaire classique utilise des câbles Ethernet pour relier des ordinateurs et des périphériques via un routeur ou un commutateur.
-Resau sans fils: c'est un réseau informatique numérique qui connecte différents postes ou systèmes entre eux par ondes radio.
5-4-Le type d'organisation :
-Reseau porte a postes : chaque ordinateur est libre de partages des donnees,donc chaque ordinateur est un serveur/client.
-Réseau client serveur : la majorité sont des stations postes clients, les autres sont des serveurs.

6-Type de Typologie: il existe trois type en bus , anneau , etoile , maillée:
-Les reseaux en bus : les ordinateur sont connecté en serie sur le bus par l'intermediaire d'un connecteur BNC , est reliée par un cable coaxial .
-les reseaux en etoile :les equipements de réseau sont reliées a un systeme materiel central, pour assurer la communication entre les déffirents équipements de reseau.
-Les reseaux en anneau : Toutes les entités sont reliées entre elles dans une boucle fermée,les données circulent dans une direction unique d'une entité a la suivante.
-Maillé: tous les noeuds de réseau sont connecté les uns avsc les autres.

Chapitre 3.
Le modèle OSI

1 ) Les composants matériel d'un Réseau informatique

les cartes réseau :

:Les cartes réseau sont installées à l'intérieur de l'ordinateur et sont généralement connectées à une prise Ethernet ou Wi-Fi. Les cartes réseau jouent un rôle essentiel dans la communication entre ordinateurs sur un réseau.

Switch :

Un switch est un composant matériel utilisé pour connecter des appareils sur un réseau local . Lorsqu'un appareil se connecte au switch, le switch vérifie son adresse MAC et stocke cette information dans une table de correspondance. Les switches sont disponibles en différentes tailles, allant des petits switches pour les réseaux domestiques et les petits bureaux, aux switches de niveau entreprise pour les réseaux plus importants.

HUB:

Un hub est un composant matériel utilisé pour connecter plusieurs appareils sur un réseau local . Le hub fonctionne comme un répéteur, il reçoit les données envoyées par un appareil et les transmet à tous les autres appareils connectés. Les hubs ont été largement remplacés par des switches, qui sont plus efficaces pour la gestion du trafic réseau et permettent une communication plus rapide et plus sécurisée entre les appareils connectés.

2 )Le Modèle OSI:

OSI (Open Systems Interconnection): est compose de sept couches Chaque couche est responsable de fonctions spécifiques liées à la communication de données entre les appareils de réseau. Le modèle OSI est souvent utilisé comme référence pour la conception et la mise en œuvre de réseaux informatiques.Les sept couches du modèle OSI sont les suivantes :

  • La couche physique
  • La couche liaison de données
  • La couche réseau
  • La couche transport
  • La couche session
  • La couche présentation
  • La couche application

    • 3 ) Le Modèle TCP/IP:

    Le modèle TCP/IP est un modèle de référence pour les communications de réseau, qui est utilisé pour la conception et la mise en œuvre des réseaux informatiques modernes.le modèle TCP/IP divise les communications en quatre couches principales:

  • La couche liaison de données
  • La couche réseau
  • La couche transport
  • La couche application

    • Le modèle TCP/IP est un modèle de référence pour les communications de réseau, qui est utilisé pour la conception et la mise en œuvre des réseaux informatiques modernes.

    4 ) L’adressage:

    Les communications dans un réseau obéissent à des règles :

  • Adressage qui permet d'identifier de manière unique les deux unités en communication
  • Architecture qui définit les rôles endossés par les deux unités
  • Protocoles qui assurent l'échange des données
  • C’est quoi un adressage IP : Une adresse IP (Internet Protocol) est le numéro qui identifie chaque ordinateur dans les réseaux qu’il appartient.

    • 5 ) L’adresse IPV4:

    Les 4 octets composent deux parties une partie pour Partie réseau et l’autre pour la partie hôtes. Les différents types d’adresse IPV4: l’adressage IPV4 s’effectué selon les types suivants :

  • L’adresse réseau.
  • L’adresse de diffusion.
  • L’adresse hôtes.
  • Le masque de réseau et de sous réseaux

    1) Introduction

    nous allons défenir ce que sont Git et Git hub également on va comprendre porquoi Git est si populaire et les systéme de gestion de version sont incontournable

    Qu'est que Git

    Git est un système de controle de version distribué largement utilisé pour gérer les version et les modifications du code source d'un projet de fusionner leur modifications et de revenir à des version antérieures si nécessair. Git facilite également la collaboration entre les membres d'une équipe de dévelopement en permettant le partage et la synchronisation du code source depot de distance tel que github

    2) A quoi sert concrètement un système de gestion de version

    un système de gestion de version (SGV) sert à suivre et controler les modifications apportées aux fichiers d'un projet informatique au fil du temps.Il permet de garder un historique détaillé des changements effectués de collaborer efficacement avec d'autre dévoellopeurs de resoudre les conflits lors de la fusion de modifications en tout un SGV facilite la gestion du code source et améliore la productivité et la fiabilité du développement logiciel

    3) Les deux modèles des logiciels de gestion de version : modèle centralisé vs modèle décentralisé

    Modèle centralisé : Dans ce modèle, il existe un référentiel central qui stocke l'ensemble du code source et de l'historique des modifications. Les développeurs doivent se connecter à ce référentiel central pour effectuer des different opérations

    Modèle décentralisé : Dans ce modèle, chaque développeur possède une copie complète du référentiel, y compris l'historique modifications. Cela permet aux développeurs de travailler en mode hors ligne et de soumettre leurs modifications à leur referentiel local.ce qui donne des aventage notable sur le model centralisé dans la simplisité et flexibilité de travaille et securité.

    GitHub est une plateforme de développement logiciel basée sur le cloud qui offre des fonctionnalités de gestion de version et de collaboration. C'est l'un des services d'hébergement de code source les plus populaires utilisés par les développeurs du monde entier. Il permet aux développeurs de télécharger, stocker et gérer leurs référentiels de code source

    Pour récapituler GitHub est un service en ligne d’hébergement de dépôts Git qui fait office de serveur central pour ces depot

    9) La gestion des informations selon Git

    Git gère les informations de versionnement à l'aide des concepts suivants :

    •Référentiel (Repository) : L'espace où sont stockées toutes les informations de versionnement.

    •Index (Staging Area) : Une zone intermédiaire entre le répertoire de travail et le référentiel, utilisée pour sélectionner les modifications à inclure dans le prochain commit

    •Commit : Un instantané des modifications effectuées dans le répertoire de travail, accompagné d'un message descriptif.

    •Branches : Des lignes de développement isolées permettant de travailler sur différentes fonctionnalités ou corrections

    •Fusion (Merge) : La combinaison des modifications de deux branches distinctes. Pousser (Push) : L'envoi des modifications locales vers un référentiel distant.

    •Tirer (Pull) : La récupération des modifications d'un référentiel distant pour synchroniser le référentiel local.

    10) Les états des fichiers

    Git suit les modifications des fichiers en utilisant les états de suivi et d'indexation. Un fichier peut être soit "suivi" s'il fait partie du dernier instantané enregistré dans Git, soit "non suivi" s'il n'appartient pas au dernier instantané et n'a pas été indexé.

    Chaque fichier suivi peut avoir l'un des trois états suivants : modifié (modified), indexé (staged) ou validé (committed). Lorsque vous démarrez un dépôt Git à partir d'un dépôt local, vous demandez à Git de valider tous les fichiers du projet.

    Lorsque vous travaillez sur votre projet, vous ajouterez probablement de nouveaux fichiers ou modifierez des fichiers existants Les fichiers modifiés sont considérés comme modifiés par Git tant qu'ils ne sont pas indexés. L'indexation d'un fichier signifie indiquer à Git que le fichier modifié ou nouveau doit être inclus dans le prochain instantané

    Enfin, lorsque vous demandez à Git de prendre un instantané, vous enregistrez l'état actuel du projet en base de données en incluant les fichiers indexés et non modifiés. Ces fichiers font partie de l'instantané.

    11) Les zones de travail

    Git utilise trois zones de travail pour gérer les états de fichiers : le répertoire de travail (working tree), la zone d'index (staging area) et le répertoire Git (repository).

    Le répertoire de travail est l'endroit où les fichiers sont extraits de la base de données compressée de Git. C'est là que vous travaillez sur les fichiers, en effectuant des modifications ou en les créant.

    La zone d'index est un fichier situé généralement dans le répertoire Git. Elle stocke les informations sur ce qui sera inclus dans le prochain commit. Lorsque vous modifiez ou créez un fichier, vous pouvez choisir de l'ajouter à l'index.

    Le répertoire Git est l'endroit où Git stocke les méta-données et la base de données des objets de votre projet. C'est la partie centrale de Git.

    Le processus de travail avec Git consiste à travailler sur les fichiers dans le répertoire de travail. Lorsque vous modifiez ou créez un fichier, vous pouvez choisir de l'indexer en l'ajoutant à la zone d'index.

    17) Qu’est ce qu’une branche ?

    Git utilise un système de branches basé sur des pointeurs vers des commits. Créer une branche dans Git signifie simplement créer un nouveau pointeur vers un commit spécifique, ce qui est beaucoup plus léger et plus rapide que de recopier l'intégralité du répertoire detravail. La branche par défaut dans Git s'appelle "master", mais elle n'a rien de spécial et peut être renommée. en tout un branche dans un git estun pointeur léger et diplaçable vers un de ces commits,"master" pointvers le dernier commit réalisée
